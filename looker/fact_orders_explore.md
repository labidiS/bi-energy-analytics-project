# Looker Explore: fact_orders

## Source table
fact_orders

## Grain
One row per order.

## Description
This Explore exposes analytics-ready order data for business users.  
The underlying table is modeled in the warehouse to ensure correct metrics and consistent results across dashboards.

## Dimensions
- order_id  
- customer_id  
- order_status  
- order_purchase_date (derived from order_purchase_ts)

## Measures
- total_orders = COUNT(order_id)
- total_revenue = SUM(total_order_value)
- average_order_value = total_revenue / total_orders

## Filters
- order_status
- order_purchase_date

## Example business questions
- How many orders are placed per day?
- How does revenue evolve over time?
- What is the distribution of order statuses?

## Example SQL generated by Looker
```sql
SELECT
  DATE(order_purchase_ts) AS order_date,
  COUNT(order_id) AS total_orders,
  SUM(total_order_value) AS total_revenue
FROM fact_orders
GROUP BY order_date
ORDER BY order_date;
